The modal directive and service allow you to launch modals in a consistent way in SKY UX applications. The SkyModalService provider lauches the modals, while the sky-modal directive provides a common look-and-feel for modal content. Within the sky-modal directive, you have options to display a common modal header, specify the body content, and display a common modal footer and buttons. Opens a modal using the specified component. The optional `providers` parameter specifies one or more providers to be passed to the component's constructor. This can be used to pass context values from the component launching the modal to the modal component. Specifies a header for the modal. Specifies content to display in the modal's body. Specifies content to display in the modal's footer.
